## 1. 软件概述

### 1.1 软件简介

数学刷题系统是一款基于Web的在线学习平台，旨在为学生提供系统化的数学题目练习和学习管理功能。系统采用前后端分离架构，前端使用Vue3框架开发，后端使用Django REST Framework构建API服务。

### 1.2 软件特点

本系统具有以下主要特点：

1. **多学段支持**：支持小学、初中、高中、大学等多个学段的数学题目练习
2. **智能分类**：采用学段-学科-模块-章节的四级分类体系，便于题目管理和查找
3. **个性化学习**：基于用户练习记录，提供智能题目推荐和弱项分析
4. **成长系统**：包含等级、经验值、连续学习天数、成就系统等激励机制
5. **数据统计**：提供详细的练习数据统计和可视化分析
6. **打卡系统**：支持每日练习打卡，培养学习习惯

### 1.3 技术架构

**前端技术栈：**
- Vue 3.x：渐进式JavaScript框架
- Vite：快速的前端构建工具
- Vue Router：单页面应用路由管理
- Axios：HTTP客户端库

**后端技术栈：**
- Django 3.x/4.x：Python Web框架
- Django REST Framework：RESTful API开发框架
- SQLite/PostgreSQL：数据库系统
- JWT：用户认证机制

### 1.4 运行环境要求

**服务器端：**
- Python 3.8及以上版本
- Django 3.2及以上版本
- 数据库：SQLite（开发环境）或PostgreSQL/MySQL（生产环境）

**客户端：**
- 现代浏览器（Chrome、Firefox、Safari、Edge等）
- 支持JavaScript和HTML5
- 建议屏幕分辨率：1920x1080或更高

## 2. 系统架构

### 2.1 整体架构设计

系统采用前后端分离的架构模式，前端和后端通过RESTful API进行数据交互。整体架构如下图所示：

```
┌─────────────────┐
│   用户浏览器    │
│   (Vue3前端)    │
└────────┬────────┘
         │ HTTP/HTTPS
         │ RESTful API
         ▼
┌─────────────────┐
│  Django后端API  │
│  (DRF框架)      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   数据库层      │
│  (SQLite/MySQL) │
└─────────────────┘
```

### 2.2 后端架构

后端采用Django的MVC（Model-View-Controller）架构模式：

**Model层（数据模型）：**
- `CustomUser`：用户模型，扩展了Django默认用户模型
- `Problem`：题目模型，存储题目内容、答案、难度等信息
- `GradeLevel`、`Subject`、`Module`、`Chapter`：分类体系模型
- `PracticeRecordDetail`：详细练习记录模型
- `DailyStudyStats`：每日学习统计模型
- `Achievement`、`UserAchievement`：成就系统模型
- `SmartRecommendation`：智能推荐模型

**View层（视图层）：**
- `problems/views.py`：题目相关API视图
- `users/views.py`：用户相关API视图
- 使用Django REST Framework的APIView和GenericView

**Controller层（控制层）：**
- URL路由配置：`urls.py`
- 序列化器：`serializers.py`，用于数据序列化和验证

### 2.3 前端架构

前端采用组件化开发模式：

**页面组件：**
- 登录/注册页面
- 首页（Dashboard）
- 题目列表页
- 练习页面
- 个人中心
- 统计页面

**状态管理：**
- 使用Vuex或Pinia进行全局状态管理
- 存储用户信息、认证token等

**路由管理：**
- 使用Vue Router实现单页面应用路由
- 支持路由守卫，保护需要认证的页面

### 2.4 数据流设计

**用户请求流程：**
1. 用户在浏览器中操作前端界面
2. 前端通过Axios发送HTTP请求到后端API
3. 后端Django处理请求，进行业务逻辑处理
4. 后端访问数据库，进行数据查询或更新
5. 后端返回JSON格式的响应数据
6. 前端接收响应，更新界面显示

**认证流程：**
1. 用户登录，后端验证用户名和密码
2. 验证成功后，后端生成JWT token
3. 前端保存token到localStorage
4. 后续请求在HTTP Header中携带token
5. 后端验证token有效性，决定是否允许访问

## 3. 功能模块说明

### 3.1 用户管理模块

#### 3.1.1 用户注册功能

用户注册功能允许新用户创建账户。注册流程如下：

1. **注册入口**：用户可以通过以下方式进入注册页面
   - 首页右上角"注册"按钮
   - 登录页面的"注册"链接
   - 首次使用时的引导提示

2. **注册表单**：用户需要填写以下信息
   - 用户名：3-100个字符，支持字母、数字和部分特殊字符
   - 邮箱地址：必须为有效的邮箱格式
   - 密码：至少8个字符，建议包含数字和字母
   - 确认密码：必须与密码一致
   - 验证码：通过邮箱或手机号接收的6位数字验证码

3. **验证码机制**：
   - 系统生成6位随机数字验证码
   - 验证码有效期5分钟
   - 每个邮箱/手机号每天最多发送6次验证码
   - 验证码存储在数据库和缓存中，支持快速验证

4. **账号创建**：验证通过后，系统自动创建用户账号，并初始化以下数据
   - 用户唯一ID
   - 默认头像
   - 初始等级（Lv.1）
   - 初始经验值（0）
   - 学习统计数据（全部为0）

#### 3.1.2 用户登录功能

用户登录功能支持多种登录方式：

1. **账号密码登录**：
   - 输入用户名或邮箱
   - 输入密码
   - 系统验证用户凭证
   - 验证成功后生成JWT token

2. **验证码登录**（可选）：
   - 输入手机号或邮箱
   - 获取验证码
   - 输入验证码完成登录

3. **Token管理**：
   - Token有效期24小时
   - Token存储在浏览器localStorage
   - 每次API请求自动携带token
   - Token过期后需要重新登录

#### 3.1.3 用户信息管理

用户可以在个人中心查看和修改个人信息：

**可查看信息：**
- 用户名
- 邮箱地址
- 头像
- 总刷题数
- 总正确数
- 总正确率
- 当前等级
- 连续学习天数

**可修改信息：**
- 用户名（需验证唯一性）
- 邮箱地址（需验证格式和唯一性）
- 头像（支持URL或base64图片）

### 3.2 题目管理模块

#### 3.2.1 题目分类体系

系统采用四级分类体系组织题目：

**第一级：学段（GradeLevel）**
- 小学
- 初中
- 高中
- 大学
- 专业

**第二级：学科（Subject）**
- 每个学段下包含多个学科
- 例如：数学、语文、英语、计算机等

**第三级：知识模块（Module）**
- 每个学科下包含多个知识模块
- 例如：数与代数、图形与几何、数据结构、算法等

**第四级：章节（Chapter）**
- 每个模块下包含多个章节
- 例如：数的认识、一次函数、线性表、排序算法等

**知识点（KnowledgePoint）**
- 知识点与题目是多对多关系
- 一道题目可以关联多个知识点
- 用于智能推荐和统计分析

#### 3.2.2 题目类型

系统支持三种题目类型：

1. **选择题（choice）**
   - 提供多个选项供用户选择
   - 选项存储在JSON字段中
   - 正确答案为选项编号或内容

2. **填空题/问答题（text）**
   - 用户输入文本答案
   - 系统进行答案匹配（支持模糊匹配）

3. **判断题（judge）**
   - 用户选择"正确"或"错误"
   - 答案为布尔值

#### 3.2.3 题目难度等级

题目难度分为5个等级：

- 1级：入门（适合初学者）
- 2级：基础（掌握基本概念）
- 3级：标准（常规难度）
- 4级：进阶（需要深入理解）
- 5级：挑战（高难度题目）

#### 3.2.4 题目属性

每道题目包含以下属性：

- **内容（content）**：题目的题干内容
- **正确答案（correct_answer）**：标准答案
- **难度等级（difficulty_level）**：1-5级
- **题目类型（type）**：选择题/填空题/判断题
- **选项（options）**：选择题的选项列表（JSON格式）
- **解析（analysis）**：题目的详细解析和解题思路
- **预计完成时间（estimated_time）**：单位分钟
- **统计信息**：
  - 总尝试次数（total_attempts）
  - 正确次数（correct_attempts）
  - 正确率（correct_rate）：自动计算

### 3.3 练习模块

#### 3.3.1 题目获取

系统提供多种方式获取练习题目：

1. **按章节练习**：
   - 用户选择学段→学科→模块→章节
   - 系统返回该章节下的所有题目
   - 支持按难度筛选

2. **按分类练习**：
   - 用户选择题目分类（数学、数据结构、算法等）
   - 系统返回该分类下的题目
   - 支持按难度筛选

3. **智能推荐**：
   - 系统分析用户练习记录
   - 识别弱项知识点
   - 推荐相关题目进行强化练习

4. **随机练习**：
   - 系统随机抽取题目
   - 可指定题目数量和难度范围

#### 3.3.2 练习模式

系统支持两种练习模式：

1. **普通模式（normal）**：
   - 根据用户选择的难度筛选题目
   - 适合有明确学习目标的用户

2. **智能模式（smart）**：
   - 系统根据用户历史数据智能选择题目
   - 自动平衡不同难度
   - 优先推荐弱项知识点相关题目

#### 3.3.3 答题流程

用户答题的完整流程：

1. **开始练习**：
   - 用户选择练习模式、题目数量、难度等参数
   - 系统生成练习题目列表
   - 显示第一道题目

2. **答题过程**：
   - 用户阅读题目内容
   - 根据题目类型选择或输入答案
   - 可以查看题目解析（可选）
   - 记录答题用时

3. **提交答案**：
   - 用户提交答案后，系统立即判断正误
   - 显示正确答案和解析
   - 记录答题结果和用时

4. **完成练习**：
   - 所有题目完成后，显示练习总结
   - 统计总题数、正确数、正确率、总用时
   - 更新用户学习数据

#### 3.3.4 练习记录

系统详细记录每次练习的信息：

**练习记录详情（PracticeRecordDetail）包含：**
- 用户ID
- 练习日期
- 题目ID
- 是否答对
- 答题用时（秒）
- 题目难度
- 题目类型
- 所属章节
- 涉及知识点
- 答题状态（正确/错误）

**每日练习记录（DailyPracticeRecord）包含：**
- 用户ID
- 年份和月份
- 每日练习状态（JSON格式）
  - 日期：是否完成练习
  - 总题数
  - 正确数
  - 正确率
  - 用时
  - 难度

### 3.4 统计模块

#### 3.4.1 个人统计

系统提供全面的个人学习统计：

**基础统计：**
- 总刷题数：累计完成的题目总数
- 总正确数：累计答对的题目数
- 总正确率：总正确数/总刷题数×100%
- 当前等级：根据总刷题数计算
- 连续学习天数：连续打卡的天数

**本周学习概要：**
- 本周总学习时长（分钟）
- 本周总练习题数
- 本周正确题数
- 本周平均正确率

**今日学习数据：**
- 今日学习时长
- 今日练习题数
- 今日正确率

#### 3.4.2 练习趋势分析

系统提供多种趋势分析图表：

1. **练习次数趋势**：
   - 按日期统计每日练习题目数
   - 支持查看最近7天、30天、90天的数据
   - 以折线图形式展示

2. **正确率趋势**：
   - 按日期统计每日平均正确率
   - 按题型分别统计正确率
   - 支持多维度对比分析

3. **学习时长趋势**：
   - 统计每日学习时长
   - 计算平均每日学习时长
   - 识别学习高峰期

#### 3.4.3 知识点掌握度分析

系统分析用户对各知识点的掌握情况：

1. **知识点统计**：
   - 统计每个知识点的练习次数
   - 计算每个知识点的正确率
   - 识别掌握较好的知识点和薄弱知识点

2. **雷达图展示**：
   - 以雷达图形式展示知识点掌握度
   - 直观显示学习强项和弱项
   - 支持多知识点对比

3. **弱项分析**：
   - 自动识别正确率低于60%的知识点
   - 标记为弱项知识点
   - 用于智能推荐系统

#### 3.4.4 题型掌握度分析

系统分析用户对不同题型的掌握情况：

**题型分类统计：**
- 选择题：统计练习次数和正确率
- 填空题：统计练习次数和正确率
- 判断题：统计练习次数和正确率

**题型对比分析：**
- 对比不同题型的正确率
- 识别用户擅长的题型
- 识别需要加强的题型

#### 3.4.5 学习热力图

系统提供学习热力图功能：

1. **数据来源**：
   - 统计最近一年的每日练习数据
   - 以日期为横轴，以练习量为颜色深浅

2. **可视化展示**：
   - 颜色越深表示练习量越多
   - 颜色越浅表示练习量越少
   - 空白表示当天未练习

3. **使用场景**：
   - 查看学习习惯
   - 识别学习规律
   - 发现学习空档期

### 3.5 成长系统模块

#### 3.5.1 等级系统

系统根据用户总刷题数自动计算等级：

**等级划分：**
- Lv.1：总刷题数 < 80题
- Lv.2：总刷题数 80-199题
- Lv.3：总刷题数 200-399题
- Lv.4：总刷题数 ≥ 400题

**经验值计算：**
- 每完成一道题目获得1点经验值
- 当前等级经验值 = 总刷题数 % 当前等级所需经验
- 下一级所需经验根据等级不同而不同

**等级显示：**
- 在用户信息中显示当前等级
- 显示经验值进度条
- 显示距离下一级还需的经验值

#### 3.5.2 连续学习天数

系统记录用户的连续学习天数：

**计算规则：**
1. 用户完成当日练习后，系统检查昨日是否有练习记录
2. 如果昨日有练习，连续天数+1
3. 如果昨日无练习，连续天数重置为1
4. 系统记录最后活跃时间，用于判断连续性

**显示位置：**
- 个人中心显示连续学习天数
- 首页显示连续学习天数
- 打卡日历中标记连续天数

#### 3.5.3 成就系统

系统提供丰富的成就奖励机制：

**成就类型：**
1. **打卡成就**：连续学习N天
2. **正确率成就**：达到指定正确率
3. **题量成就**：完成指定数量的题目
4. **连续学习成就**：连续学习达到指定天数
5. **等级成就**：达到指定等级

**成就解锁：**
- 系统自动检测用户是否满足成就条件
- 满足条件后自动解锁成就
- 记录解锁时间
- 在成就页面显示已解锁成就

**成就展示：**
- 成就列表页面显示所有成就
- 已解锁成就显示解锁时间
- 未解锁成就显示解锁条件
- 支持按类型筛选成就

### 3.6 智能推荐模块

#### 3.6.1 推荐算法

系统基于用户练习记录进行智能推荐：

**推荐流程：**
1. **数据收集**：收集用户最近30天的练习记录
2. **知识点分析**：统计每个知识点的练习次数和正确率
3. **弱项识别**：识别正确率低于60%且练习次数≥3的知识点
4. **题目推荐**：从弱项知识点相关的题目中推荐
5. **去重处理**：排除用户已经练习过的题目
6. **结果存储**：将推荐结果存储到SmartRecommendation模型

#### 3.6.2 推荐更新机制

系统支持手动和自动更新推荐：

**手动更新：**
- 用户点击"更新推荐"按钮
- 系统重新分析用户最新练习数据
- 生成新的推荐题目列表

**自动更新（可选）：**
- 用户完成练习后自动触发
- 定期（如每天）自动更新
- 根据用户活跃度决定更新频率

#### 3.6.3 推荐结果展示

推荐题目在以下位置展示：

1. **首页推荐区**：
   - 显示推荐的题目列表
   - 显示推荐理由（如"强化弱项：一次函数"）
   - 支持一键开始练习

2. **个人中心**：
   - 显示弱项知识点列表
   - 显示推荐题目数量
   - 支持查看详细推荐

3. **练习页面**：
   - 在练习模式选择中提供"智能推荐"选项
   - 选择后自动加载推荐题目

系统遵循RESTful API设计规范：

**URL设计：**
- 使用名词表示资源，如`/api/problems/`
- 使用HTTP方法表示操作：
  - GET：获取资源
  - POST：创建资源
  - PUT：更新资源
  - DELETE：删除资源

**响应格式：**
```json
{
  "success": true,
  "status": 200,
  "message": "操作成功",
  "data": {
    // 具体数据
  }
}
```

**错误响应格式：**
```json
{
  "success": false,
  "status": 400,
  "message": "错误信息",
  "errors": {
    // 详细错误信息
  }
}
```

- 路径：`POST /api/users/register/`
- 请求体：`{username, email, password1, password2, verification_code}`

- 路径：`POST /api/users/login/`

- 路径：`POST /api/users/update-profile/`
- 认证：需要JWT token
- 请求体：`{username?, email?, avatar?}`
- 功能：更新用户个人信息

**发送验证码API：**
- 路径：`POST /api/users/send-verification-code/`
- 请求体：`{email, type}`
- 功能：发送验证码到邮箱或手机

**验证验证码API：**
- 路径：`POST /api/users/verify-code/`

### 5.3 认证与授权

#### 5.3.1 JWT认证机制

系统使用JWT（JSON Web Token）进行用户认证：

**Token生成：**
```python
payload = {
    'user_id': user.id,
    'username': user.username,
    'exp': datetime.now(timezone.utc) + timedelta(hours=24),
    'iat': datetime.now(timezone.utc),
}
token = jwt.encode(payload, settings.SECRET_KEY, algorithm='HS256')
```

**Token验证：**
- 从HTTP Header中提取token：`Authorization: Bearer <token>`
- 使用SECRET_KEY验证token签名
- 检查token是否过期
- 提取用户信息

**Token存储：**
- 前端：存储在localStorage
- 后端：不存储token，每次请求验证

#### 5.3.2 权限控制

系统使用Django REST Framework的权限类：

**AllowAny：** 允许匿名访问
- 题目列表、题目详情等公开API

**IsAuthenticated：** 需要认证
- 用户信息、练习记录、统计数据等

**自定义权限：** 可根据需要扩展
- 如：IsOwner（只能访问自己的数据）

## 7. 常见问题

### 7.1 安装与运行问题

**Q1: 如何安装依赖？**

A: 后端使用`pip install -r requirements.txt`，前端使用`npm install`或`yarn install`。

**Q2: 数据库迁移失败怎么办？**

A: 检查数据库连接配置，确保数据库文件权限正确。可以尝试删除迁移文件重新生成。

**Q3: 端口被占用怎么办？**

A: 后端可以指定其他端口：`python manage.py runserver 8001`。前端Vite会自动尝试下一个可用端口。

### 7.2 功能使用问题

**Q1: 如何开始第一次练习？**

A: 登录后，在首页点击"开始练习"按钮，选择题目数量和难度，点击"开始"即可。

**Q2: 如何查看我的学习统计？**

A: 点击导航栏"个人中心"或"统计"，可以查看详细的学习统计数据。

**Q3: 如何更新智能推荐？**

A: 完成练习后系统会自动更新，也可以手动点击"更新推荐"按钮。

**Q4: 忘记密码怎么办？**

A: 可以使用验证码登录功能，或联系管理员重置密码。

### 7.3 技术问题

**Q1: Token过期怎么办？**

A: Token有效期为24小时，过期后需要重新登录。系统会自动检测并跳转到登录页。

**Q2: CORS错误如何解决？**

A: 检查后端`settings.py`中的CORS配置，确保允许前端域名访问。

**Q3: 验证码收不到怎么办？**

A: 开发环境下验证码会在控制台打印。生产环境需要配置邮件或短信服务。

### 7.4 数据问题

**Q1: 如何导入题目数据？**

A: 可以使用Django管理后台导入，或运行`python run_population.py`脚本生成示例数据。

**Q2: 统计数据不准确怎么办？**

A: 检查练习记录是否正确保存，可以手动触发统计更新。

**Q3: 如何备份数据？**

A: 使用Django的`dumpdata`命令导出数据，或直接备份数据库文件。

## 8. 系统功能详细说明

### 8.1 用户注册功能详细流程

#### 8.1.1 注册页面布局

注册页面采用响应式设计，包含以下元素：

1. **页面头部**：显示系统Logo和名称
2. **注册表单区域**：居中显示，包含所有输入字段
3. **表单字段**：
   - 用户名输入框：带占位符提示"请输入3-100个字符的用户名"
   - 邮箱输入框：带占位符提示"请输入有效的邮箱地址"
   - 密码输入框：带占位符提示"至少8个字符"
   - 确认密码输入框：带占位符提示"请再次输入密码"
   - 验证码输入框：带占位符提示"请输入6位验证码"
   - 获取验证码按钮：点击后发送验证码
4. **服务条款复选框**：必须勾选才能注册
5. **提交按钮**：注册按钮，点击后提交表单
6. **登录链接**：已有账号可直接登录

#### 8.1.2 前端验证流程

在用户提交表单前，前端进行实时验证：

1. **用户名验证**：
   - 检查长度是否在3-100字符之间
   - 检查字符是否合法（字母、数字、部分特殊字符）
   - 实时显示验证结果

2. **邮箱验证**：
   - 检查邮箱格式是否正确
   - 使用正则表达式验证：`^[^\s@]+@[^\s@]+\.[^\s@]+$`
   - 实时显示格式错误提示

3. **密码验证**：
   - 检查密码长度是否至少8个字符
   - 检查密码强度（可选：包含数字和字母）
   - 实时显示密码强度指示器

4. **确认密码验证**：
   - 检查两次输入的密码是否一致
   - 实时显示匹配状态

5. **验证码验证**：
   - 检查验证码是否为6位数字
   - 检查验证码是否已输入

#### 8.1.3 后端处理流程

后端接收到注册请求后的处理流程：

1. **数据接收**：
   - 从请求体中提取JSON数据
   - 解析用户名、邮箱、密码、验证码等字段

2. **数据验证**：
   - 使用Django表单验证器验证数据格式
   - 检查用户名唯一性：查询数据库是否已存在
   - 检查邮箱唯一性：查询数据库是否已注册
   - 验证密码强度：检查是否符合安全要求

3. **验证码验证**：
   - 从缓存或数据库中查找验证码
   - 检查验证码是否正确
   - 检查验证码是否过期（5分钟有效期）
   - 检查验证码是否已被使用
   - 验证通过后标记验证码为已使用

4. **用户创建**：
   - 如果所有验证通过，创建新用户对象
   - 使用Django的密码哈希函数加密密码
   - 设置用户默认属性：
     * 总刷题数：0
     * 总正确数：0
     * 总正确率：0.0
     * 当前等级：1
     * 总经验值：0
     * 连续学习天数：0
     * 默认头像：系统默认头像URL
   - 保存用户到数据库

5. **响应返回**：
   - 创建成功：返回200状态码和用户ID
   - 创建失败：返回400状态码和详细错误信息

#### 8.1.4 错误处理机制

系统对注册过程中的各种错误情况进行处理：

1. **用户名已存在**：
   - 错误信息："用户名已被使用"
   - 提示用户更换用户名或直接登录

2. **邮箱已注册**：
   - 错误信息："邮箱已被注册"
   - 提供"登录"和"忘记密码"链接

3. **验证码错误**：
   - 错误信息："验证码错误"
   - 提示用户重新获取验证码

4. **验证码过期**：
   - 错误信息："验证码已过期，请重新获取"
   - 自动清除过期验证码

5. **密码不符合要求**：
   - 错误信息："密码至少需要8个字符"
   - 显示密码强度要求

6. **网络错误**：
   - 错误信息："网络连接失败，请检查网络"
   - 提示用户稍后重试

### 8.2 用户登录功能详细流程

#### 8.2.1 登录方式

系统支持两种登录方式：

**方式一：账号密码登录**

1. **输入用户名或邮箱**：
   - 系统支持使用用户名或邮箱登录
   - 自动识别输入的是用户名还是邮箱格式
   - 如果输入的是邮箱格式，使用邮箱字段查询
   - 如果输入的不是邮箱格式，使用用户名字段查询

2. **输入密码**：
   - 密码输入框支持显示/隐藏切换
   - 使用加密传输（HTTPS）

3. **记住我选项**（可选）：
   - 勾选后，token有效期延长至7天
   - 未勾选，token有效期24小时

**方式二：验证码登录**（可选功能）

1. **输入手机号或邮箱**：
   - 输入手机号或邮箱地址
   - 系统验证格式

2. **获取验证码**：
   - 点击"获取验证码"按钮
   - 系统发送6位数字验证码
   - 验证码有效期5分钟

3. **输入验证码**：
   - 输入收到的6位验证码
   - 系统验证验证码

#### 8.2.2 认证流程

后端认证处理流程：

1. **接收登录请求**：
   - 从请求体中提取用户名/邮箱和密码或验证码

2. **用户验证**：
   - 使用Django的authenticate函数验证用户
   - 对于密码登录：使用check_password验证密码
   - 对于验证码登录：验证验证码有效性

3. **生成JWT Token**：
   - 如果验证成功，生成JWT token
   - Token包含以下信息：
     * user_id：用户ID
     * username：用户名
     * exp：过期时间（24小时后）
     * iat：签发时间
   - 使用SECRET_KEY签名token

4. **返回响应**：
   - 成功：返回200状态码、token和用户基本信息
   - 失败：返回400状态码和错误信息

#### 8.2.3 Token管理

前端Token管理机制：

1. **Token存储**：
   - 登录成功后，将token存储到localStorage
   - 键名：`auth_token`或`access_token`
   - 同时存储用户基本信息（可选）

2. **Token使用**：
   - 每次API请求时，从localStorage读取token
   - 在HTTP Header中添加：`Authorization: Bearer <token>`
   - 使用Axios拦截器自动添加

3. **Token验证**：
   - 后端接收到请求后，从Header提取token
   - 验证token签名和有效期
   - 如果token有效，提取用户信息
   - 如果token无效或过期，返回401错误

4. **Token刷新**（可选）：
   - 系统可以支持token刷新机制
   - 在token即将过期时，使用refresh token获取新token
   - 延长用户登录状态

5. **Token清除**：
   - 用户退出登录时，清除localStorage中的token
   - 清除用户信息缓存

### 8.3 题目管理详细说明

#### 8.3.1 题目创建流程

管理员或教师创建题目的完整流程：

1. **选择分类**：
   - 首先选择学段（小学/初中/高中/大学）
   - 然后选择学科（数学/语文等）
   - 接着选择知识模块（数与代数/图形与几何等）
   - 最后选择章节（具体章节名称）

2. **填写题目内容**：
   - 题目题干：使用富文本编辑器输入
   - 支持数学公式（LaTeX格式）
   - 支持图片插入
   - 支持代码块（如果是编程题）

3. **设置题目类型**：
   - 选择题：需要添加选项（至少2个，通常4个）
   - 填空题：可以设置多个空，每个空有独立答案
   - 判断题：只需设置正确答案（正确/错误）

4. **设置正确答案**：
   - 选择题：输入正确选项的编号或内容
   - 填空题：输入每个空的正确答案，用分隔符分隔
   - 判断题：输入"正确"或"错误"

5. **添加选项**（仅选择题）：
   - 输入选项A的内容
   - 输入选项B的内容
   - 输入选项C的内容
   - 输入选项D的内容
   - 可以添加更多选项（E、F等）

6. **编写题目解析**：
   - 详细说明解题思路
   - 列出解题步骤
   - 说明涉及的知识点
   - 提供相关公式或定理

7. **设置题目属性**：
   - 难度等级：选择1-5级
   - 预计完成时间：输入分钟数
   - 关联知识点：选择相关知识点（可多选）
   - 标签：添加题目标签（如"重点"、"易错"等）

8. **保存题目**：
   - 点击"保存"按钮
   - 系统验证题目完整性
   - 保存到数据库
   - 返回题目ID

#### 8.3.2 题目编辑流程

编辑已有题目的流程：

1. **查找题目**：
   - 在题目列表中找到要编辑的题目
   - 或通过搜索功能查找
   - 点击"编辑"按钮

2. **加载题目数据**：
   - 系统从数据库加载题目信息
   - 填充到编辑表单中

3. **修改内容**：
   - 可以修改题目的任何字段
   - 实时保存草稿（可选功能）

4. **保存修改**：
   - 点击"保存"按钮
   - 系统更新数据库记录
   - 更新修改时间戳

#### 8.3.3 题目审核机制

系统可以设置题目审核流程：

1. **题目状态**：
   - 草稿：未完成的题目
   - 待审核：已提交等待审核
   - 已审核：审核通过，可以发布
   - 已拒绝：审核未通过，需要修改

2. **审核流程**：
   - 教师创建题目后，状态为"待审核"
   - 管理员审核题目
   - 审核通过后，状态改为"已审核"，题目可以显示
   - 审核未通过，返回修改意见

3. **审核标准**：
   - 题目内容是否完整
   - 答案是否正确
   - 解析是否详细
   - 难度等级是否合理

   - 转换为小写（可选，根据题目要求）

   - 支持模糊匹配（如"2"和"2.0"都算正确）

3. **判断题判断**：
   - 用户选择与正确答案比较
   - 布尔值比较

4. **判断结果**：
   - 立即显示判断结果
   - 正确：显示"✓ 回答正确"
   - 错误：显示"✗ 回答错误"

### 8.6 智能推荐系统详细说明

#### 8.6.1 推荐算法原理

系统使用的推荐算法基于协同过滤和内容过滤：

1. **数据收集**：
   - 收集用户最近30天的练习记录
   - 记录每道题的答题情况
   - 记录答题用时和正确性

2. **知识点分析**：
   - 统计每个知识点的练习次数
   - 计算每个知识点的正确率
   - 识别薄弱知识点（正确率<60%且练习次数≥3）

3. **题目推荐**：
   - 从薄弱知识点相关的题目中选择
   - 排除用户已练习过的题目
   - 优先推荐中等难度的题目
   - 限制推荐数量（通常10-20题）

4. **推荐更新**：
   - 用户完成练习后自动更新
   - 或定期（如每天）自动更新
   - 或用户手动触发更新

#### 8.6.2 推荐策略

系统采用多种推荐策略：

1. **弱项强化策略**：
   - 重点推荐薄弱知识点的题目
   - 帮助用户针对性练习
   - 提高整体学习效果

2. **难度递进策略**：
   - 根据用户当前水平推荐合适难度的题目
   - 避免过难或过易的题目
   - 逐步提升难度

3. **多样性策略**：
   - 确保推荐题目的多样性
   - 避免重复推荐相同类型的题目
   - 覆盖多个知识点

4. **时效性策略**：
   - 优先推荐新题目
   - 避免推荐过时的题目
   - 保持推荐内容的新鲜度

#### 8.6.3 推荐效果评估

系统可以评估推荐效果：

1. **用户反馈**：
   - 用户对推荐题目的评分
   - 用户是否完成推荐题目
   - 用户是否收藏推荐题目

2. **学习效果**：
   - 推荐题目的正确率
   - 用户知识点掌握度的提升
   - 整体学习数据的改善

3. **算法优化**：
   - 根据反馈调整推荐算法
   - 优化推荐参数
   - 提高推荐准确性

### 8.7 成长系统详细说明

#### 8.7.1 等级计算规则

系统根据用户总刷题数计算等级：

**等级1（新手）**：
- 条件：总刷题数 < 80题
- 当前等级经验值 = 总刷题数
- 下一级所需经验 = 80题
- 等级描述：刚刚开始学习之旅

**等级2（入门）**：
- 条件：总刷题数 80-199题
- 当前等级经验值 = (总刷题数 - 80) % 120
- 下一级所需经验 = 120题
- 等级描述：已经掌握基础知识

**等级3（进阶）**：
- 条件：总刷题数 200-399题
- 当前等级经验值 = (总刷题数 - 200) % 200
- 下一级所需经验 = 200题
- 等级描述：学习能力显著提升

**等级4（专家）**：
- 条件：总刷题数 ≥ 400题
- 当前等级经验值 = (总刷题数 - 400) % 300
- 下一级所需经验 = 300题（可无限升级）
- 等级描述：学习达人，持续进步

#### 8.7.2 经验值获取

用户可以通过以下方式获得经验值：

1. **完成题目**：
   - 每完成一道题目获得1点经验值
   - 无论答对答错都获得经验值
   - 鼓励用户多练习

2. **答对题目**（可选）：
   - 答对题目额外获得经验值
   - 答对获得2点，答错获得1点
   - 激励用户认真答题

3. **连续打卡**（可选）：
   - 连续打卡达到指定天数获得额外经验
   - 连续7天：额外50经验
   - 连续30天：额外200经验

4. **完成成就**（可选）：
   - 解锁成就获得经验奖励
   - 不同成就奖励不同经验值

#### 8.7.3 成就系统设计

系统设计多种成就类型：

**打卡成就**：
- "初出茅庐"：连续学习3天
- "坚持不懈"：连续学习7天
- "持之以恒"：连续学习30天
- "学习达人"：连续学习100天

**正确率成就**：
- "精准射手"：单次练习正确率100%
- "稳定发挥"：连续10次练习正确率≥80%
- "进步之星"：正确率提升20%

**题量成就**：
- "初试锋芒"：完成10题
- "小有成就"：完成50题
- "渐入佳境"：完成100题
- "题海遨游"：完成500题
- "刷题之王"：完成1000题

**等级成就**：
- "新手入门"：达到等级2
- "进阶之路"：达到等级3
- "专家之路"：达到等级4

**特殊成就**：
- "夜猫子"：在晚上10点后完成练习
- "早起鸟"：在早上6点前完成练习
- "周末战士"：周末完成练习

#### 8.7.4 成就解锁机制

成就解锁的完整流程：

1. **条件检测**：
   - 系统定期检测用户是否满足成就条件
   - 在用户完成相关操作后立即检测
   - 使用后台任务定期批量检测

2. **成就解锁**：
   - 检测到满足条件后，自动解锁成就
   - 创建UserAchievement记录
   - 记录解锁时间

3. **通知用户**：
   - 解锁成就时显示通知
   - 显示成就图标和名称
   - 显示获得的奖励（如经验值）

4. **成就展示**：
   - 在个人中心显示所有成就
   - 已解锁成就显示解锁时间
   - 未解锁成就显示解锁条件和进度

#### 8.8.3 连续学习天数计算

系统计算连续学习天数的算法：

1. **检查昨日记录**：
   - 用户完成今日练习后
   - 系统检查昨日是否有练习记录
   - 查询DailyPracticeRecord表

2. **更新连续天数**：
   - 如果昨日有练习：连续天数 = 原连续天数 + 1
   - 如果昨日无练习：连续天数 = 1（重新开始）
   - 更新用户表的streak_days字段

3. **特殊情况处理**：
   - 如果用户多天未练习，连续天数重置为1
   - 如果用户在同一天多次练习，只算一次打卡
   - 考虑时区问题，使用服务器时区

### 8.9 数据安全与隐私保护

#### 8.9.1 数据加密

系统采用多种加密措施保护数据安全：

   - 防止token被篡改

#### 8.9.3 隐私保护

系统保护用户隐私：

1. **数据最小化**：
   - 只收集必要的数据
   - 不收集敏感个人信息
   - 定期清理过期数据

2. **访问控制**：
   - 用户只能访问自己的数据

   - 统计数据匿名化处理

   - 符合隐私保护法规
